## 一、穩定性不穩定根源

### 1.1 BGK 不穩定機制

BGK 碰撞算子使用**單一鬆弛時間** τ 鬆弛所有矩：

```
f_i* = f_i - (1/τ)(f_i - f_i^eq)
```

分佈函數包含多階非平衡貢獻：

```
f_i = f_i^eq + f_i^(1) + f_i^(2) + f_i^(3) + ...
       ↑         ↑          ↑          ↑
     density   stress     heat flux   ghost modes
     momentum  tensor     (3rd order)  (no physics)
     (0th,1st)  (2nd)
```

- **f^(1)** (二階矩 / 應力張量)：對應物理黏度，鬆弛率 = 1/τ
- **f^(2)** (三階矩)：含熱通量等，部分有物理意義
- **f^(3+)** (ghost modes)：**無物理對應**，但 BGK 以相同 τ 鬆弛

當 τ → 0.5 (高 Re 或小 dt)：
- 所有非平衡模態的衰減率 → 0
- Ghost modes **不衰減** → 數值累積 → f_α < 0 → 發散

### 1.2 Ma 穩定極限的物理意義

對 D3Q19 BGK，分佈函數正定性 (f_α ≥ 0) 要求：

```
Ma_limit = c_s × sqrt(2(2τ - 1) / 3)

其中 c_s = 1/sqrt(3) (D3Q19/D3Q27 聲速)
```

| τ | Ma_limit (BGK) | 備註 |
|---|---------------|------|
| 0.505 | 0.033 | 極高 Re，幾乎無法使用 |
| 0.55 | 0.115 | |
| 0.60 | 0.211 | ← **我們的 Re=700 壁面附近** (τ≈0.604) |
| 0.65 | 0.283 | |
| 0.70 | 0.346 | |
| 0.80 | 0.461 | |
| 1.00 | 0.577 | τ=1 理論最大值 = c_s |

**關鍵洞察**：τ=0.604 時 Ma_limit ≈ 0.215，而我們正常運行 Ma ≈ 0.10。
安全係數僅 2.15×，局部瞬態峰值即可觸及。

### 1.3 自持性不穩定 (Self-Sustaining Instability)

一旦 Ma > Ma_limit：

```
f_α < 0 → ρ 計算錯誤 → u 偏離 → Ma 進一步增大 → 更多 f_α < 0
         ↑                                              ↓
         └──────────── 正反饋迴路 ────────────────────────┘
```

**此過程獨立於外力驅動**——即使將外力歸零，動能已在流場中，
碰撞算子的 ghost mode 放大足以自行驅動指數爆炸。

這就是為什麼 Force Controller (FREEZE) 無法挽救已觸發的 BGK 不穩定。

### 2.3 各碰撞算子穩定化機制

| 算子 | 穩定化機制 | 副作用 |
|-----|---------|------|
| **MRT** | Ghost mode 獨立鬆弛 (但 d'Humières 預設值保守) | 可能引入各向異性耗散 |
| **MRT-opt** | Ghost rates → 1.8~1.98 (接近極限) | 對三階矩過度阻尼 |
| **R-BGK** | 碰撞前重建 f：只保留 feq + 應力張量 neq，丟棄所有 ghost modes | 三階以上 neq 資訊丟失 |

| **CUM** | 在 cumulant space 碰撞，27 個 cumulant 獨立鬆弛 + λ 正則化 | 需要 D3Q27 |

## 三、Regularized LBM 詳細理論

### 3.1 Standard Regularization (Latt & Chopard 2006)

**核心公式**：

```
Step 1: 計算非平衡應力張量 (6 個獨立分量)
   Π_αβ^neq = Σ_q  e_qα · e_qβ · (f_q - f_q^eq)

Step 2: 從 Π^neq 重建一階非平衡
   f_i^(1) = w_i / (2·c_s⁴) × Σ_αβ (e_iα·e_iβ - c_s²·δ_αβ) × Π_αβ^neq

Step 3: 組合
   f_i^reg = f_i^eq + f_i^(1)

Step 4: 碰撞 (使用 f^reg 取代 f)
   f_i* = f_i^reg - (1/τ)(f_i^reg - f_i^eq)
        = f_i^eq + (1 - 1/τ) · f_i^(1)
```

**D3Q19 的 c_s⁴ = 1/9，所以 1/(2·c_s⁴) = 9/2 = 4.5**

展開 Π^neq 的 6 個獨立分量 (3D)：

```
Π_xx = Σ_q  e_qx² · (f_q - f_q^eq)
Π_yy = Σ_q  e_qy² · (f_q - f_q^eq)
Π_zz = Σ_q  e_qz² · (f_q - f_q^eq)
Π_xy = Σ_q  e_qx·e_qy · (f_q - f_q^eq)
Π_xz = Σ_q  e_qx·e_qz · (f_q - f_q^eq)
Π_yz = Σ_q  e_qy·e_qz · (f_q - f_q^eq)
```

重建公式 (每個方向 q)：

```
f_q^(1) = w_q × 4.5 × [
    (e_qx² - 1/3) × Π_xx +
    (e_qy² - 1/3) × Π_yy +
    (e_qz² - 1/3) × Π_zz +
    2·e_qx·e_qy × Π_xy +
    2·e_qx·e_qz × Π_xz +
    2·e_qy·e_qz × Π_yz
]
```

**FLOP 計算**：
- 計算 6 個 Π 分量：19×6 = 114 FMA + 6 減法
- 重建 19 個 f^(1)：19×12 = 228 FMA
- 總計 ~350 FLOPs/point（與 MRT 矩陣乘法相當）

### 3.2 Recursive Regularization — RR (Malaspinas 2015)

在 Standard Reg 基礎上加入二階 Chapman-Enskog 修正：

```
f_i^reg = f_i^eq + f_i^(1) + f_i^(2,CE)

f_i^(2,CE) 需要 Π^neq 的空間梯度：∂Π_αβ/∂x_γ
```

**在 GILBM 曲線座標中的考量**：
- 梯度計算需要度量項轉換：`∂Π/∂x = ∂Π/∂η · ∂η/∂x + ∂Π/∂ξ · ∂ξ/∂x + ∂Π/∂ζ · ∂ζ/∂x`
- 需要鄰點 Π^neq 值（stencil access）
- **工程量較大，暫不推薦首選**

### 3.3 Hybrid Recursive Regularization — HRR (Jacob et al. 2018)

```
f_i^neq_hybrid = σ · f_i^(1,reconstructed) + (1 - σ) · (f_i - f_i^eq)
                  ↑                            ↑
              從 Π^neq 重建            原始非平衡 (含 ghost modes)
```

| σ 值 | 行為 | 穩定性 | 精度 |
|------|------|--------|------|
| 0.0 | 標準 LBM (無正則化) | 最低 | 最高 (保留所有資訊) |
| 0.5 | 混合 | 中等 | 中等 |
| 0.8 | 接近完全正則化 | 高 | 略有損失 |
| 1.0 | 完全正則化 (= R-BGK) | 最高 | 最低 (丟棄 ghost modes) |

**推薦策略**：從 σ=1.0 開始確認穩定，再逐步降低 σ 找最小必要值。
Re=700 DNS 等級網格上，σ=1.0 的精度損失通常可忽略。

### 3.4 正則化與 MRT 的等價性

正則化等價於特定的 MRT 鬆弛率選擇：

```
MRT moment space (D3Q19, d'Humières 2002):
  s_0 = 0        (density, conserved)
  s_1,2,3 = 0    (momentum, conserved)
  s_9,11,13,14,15 = 1/τ  (stress tensor → viscosity)
  s_4,6,8 = s_e  (energy flux modes)
  s_16,17,18 = s_q (ghost modes)
  s_10,12 = s_ε  (energy square modes)

Standard Regularization ≡ MRT with:
  s_stress = 1/τ          (unchanged)
  s_all_others = 1.0      (immediate equilibration)

MRT d'Humières defaults:
  s_e = 1.19, s_ε = 1.4, s_q = 1.5  (< 1.0, 比正則化弱)
```

**所以正則化 > MRT tuning 的原因**：正則化將所有非物理矩的鬆弛率推到 1.0，
而 MRT tuning 最多推到 ~1.98（超過 2.0 會造成另一種不穩定）。

## 四、在 GILBM 4-Step 框架中的實現方案

### 4.1 方案 A：Step 1.75 插入正則化 (最直接)

```
Step 1:     Lagrange 插值 + Streaming → f_new_ptrs[q][index]
Step 1.5:   宏觀量 (ρ,u,v,w) → feq_d[q][index]

  ┌─────────────────────────────────────────────────────┐
  │ Step 1.75: Regularization (NEW)                      │
  │                                                      │
  │ // 1. 計算非平衡應力張量                                │
  │ Π_xx = Σ_q e_qx² · (f_new[q] - feq[q])             │
  │ Π_yy = Σ_q e_qy² · (f_new[q] - feq[q])             │
  │ Π_zz = Σ_q e_qz² · (f_new[q] - feq[q])             │
  │ Π_xy = Σ_q e_qx·e_qy · (f_new[q] - feq[q])         │
  │ Π_xz = Σ_q e_qx·e_qz · (f_new[q] - feq[q])         │
  │ Π_yz = Σ_q e_qy·e_qz · (f_new[q] - feq[q])         │
  │                                                      │
  │ // 2. 重建並覆寫 f_new                                │
  │ for q = 0..18:                                       │
  │   f_neq_reg = w[q]*4.5*((ex²-1/3)*Π_xx + ...)      │
  │   f_new[q][idx] = feq[q][idx] + f_neq_reg           │
  │                                                      │
  │ // feq_d 和 ρ/u/v/w 不變 (宏觀量守恆)                │
  └─────────────────────────────────────────────────────┘

Step 2:     重估 R_AB (使用正則化後的 f_new)
Step 3:     BGK/MRT 碰撞 → f_pc
```

**優勢**：
- Step 2 讀取正則化後的 f_new → R_AB 更穩定
- Step 3 碰撞輸入更乾淨 → f_pc 不含 ghost mode artifacts
- 宏觀量不受影響（正則化保守 ρ, ρu）

**注意**：正則化不改變宏觀量（0 階和 1 階矩守恆），只改變高階矩。
因此 feq、rho_out、u/v/w_out 不需要重新計算。

### 4.2 方案 B：碰撞步直接用 R-BGK (取代 MRT)

```
Step 2+3 合併為:
  f_re = feq_B + (f_B - feq_B) × R_AB     // 重估 (不變)

  // 正則化碰撞 (取代 MRT)
  Π^neq = 從 f_re 計算應力張量
  f_neq_reg = 從 Π^neq 重建
  f_pc = feq_B + (1 - 1/τ_A) × f_neq_reg   // R-BGK 碰撞
```

**優勢**：不需要 MRT 矩陣，代碼更簡潔
**劣勢**：失去 MRT 對不同物理矩的獨立控制

### 4.3 方案 C：MRT + Regularization (雙重防護)

保留 MRT 碰撞，在 Step 1.75 加正則化：
- 正則化清除 streaming 產生的 ghost modes
- MRT 碰撞提供對物理矩的獨立鬆弛控制
- **最保守、最穩定的組合**

### 4.4 壁面 BC 的相容性

正則化不影響壁面邊界條件 (Chapman-Enskog BC)：
- CE BC 方向由 `NeedsBoundaryCondition()` 判定
- BC 方向的 f 由 `ChapmanEnskogBC()` 直接設定
- 正則化只作用於 streaming 得到的 f → BC 方向不參與

## 五、精度影響分析

### 5.1 正則化的精度損失

| 影響項目 | 損失程度 | 原因 |
|---------|--------|------|
| 平均速度 (mean U, V) | **無** | 0 階 + 1 階矩守恆 |
| 壓力 (P) | **無** | 0 階矩守恆 |
| Reynolds stress (uu, vv, ww, uv) | **極小** | 2 階矩(應力張量)保留 |
| 三階矩 (skewness) | **小** | 被投影到 CE 估算值 |
| 能譜高頻端 | **可見** | 小尺度 ghost modes 被濾除，類似 implicit LES |
| 壁面剪應力 | **極小** | 由應力張量主導 |

### 5.2 網格解析度 vs 精度損失

| 網格類型 | y+ | 正則化精度損失 | 原因 |
|---------|-----|-------------|------|
| DNS (y+<1) | <1 | 可忽略 | Ghost modes 本就無物理意義 |
| LES (y+~5) | ~5 | 很小 | 類似 implicit SGS filtering |
| VLES (y+~15) | ~15 | 小 | 正則化實際改善結果 (見 Gehrke Fig.30) |
| RANS (y+>30) | >30 | 不適用 | 需要顯式紊流模型 |

**我們的 Re=700, N_Z=128 → y+ ≈ 0.5~2 → DNS 等級 → 精度損失可忽略**

### 5.3 與 Cumulant λ 正則化的比較

Gehrke 2022 的 λ 參數本質上也是正則化：
- λ → 0 (AO)：所有 cumulant ω → parameterized (最精確、最不穩定)
- λ_def = 10⁻²：默認正則化強度
- λ → ∞：所有 cumulant ω → 1 (≈ 完全正則化)

**Table 6 (Gehrke 2022)**: Re=700 時 λ_max = 10⁶ (無穩定性限制)
→ 等價於說：**Re=700 DNS 網格上，正則化不會影響穩定性**

## 七、我們的 GILBM 特殊考量

### 7.1 正則化在曲線座標中的相容性

正則化是**純局部操作** (只涉及當前格點的 19 個 f 值)：
- 不需要座標變換
- 不需要鄰點資訊 (Standard Reg)
- 與 GILBM 的曲線座標 streaming 完全解耦

```
GILBM streaming (曲線座標, 非局部) → f_new (物理速度空間)
                                          ↓
正則化 (局部, Cartesian 速度空間) → f_reg
                                          ↓
碰撞 (局部, 與座標系無關) → f_pc
```

### 7.2 LTS (Local Time Stepping) 與正則化

R_AB = omegadt_A / omegadt_B 在重估中使用。
正則化後的 f_new 更「乾淨」→ R_AB 放大的對象更穩定 → 雙重穩定化效果。

### 7.3 記憶體影響

Standard Regularization：**零額外記憶體**。
- Π^neq 的 6 個分量是 register 變數
- f^reg 直接覆寫 f_new (in-place)
